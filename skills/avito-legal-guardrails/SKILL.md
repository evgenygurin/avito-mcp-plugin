# Avito legal guardrails (право РФ)

> **Это не юридическая консультация.** При реальных рисках — к юристу.
> Полный разбор норм и прецедентов — в `docs/avito-legal.md`.

## Overview

Парсинг Avito в РФ несёт реальные, но не абсолютные правовые риски. Четыре оси:
смежное право на БД, персональные данные, обход техсредств защиты, недобросовестная
конкуренция. Цель скила — до начала сбора отсечь опасные действия и оставить
наименее рисковый путь.

## When to Use

- **Перед** любым сбором/сохранением/экспортом данных Avito.
- Когда в выборку попадают телефоны, имена, email продавцов.
- Когда планируется массовое/систематическое/полное извлечение объявлений.
- Когда всплывает капча/антибот и есть соблазн подключить решатель.

## Guardrails (что нельзя / что можно)

| Риск | Норма | Правило |
|---|---|---|
| Копирование существенной части БД (≥10 000 элементов) | ст. 1334 ГК, компенсация до 5 млн ₽ | не извлекать существенную долю; инкрементально по `last_seen`, не полная выгрузка категории |
| Сбор ПДн (телефоны/имена) для перепродажи | 152-ФЗ | собирать только факты (цена, площадь, характеристики) |
| Обход капчи/антибота «в лоб» | ст. 272 УК | избегать капчи ротацией чистых IP, а не решать её ([scraping-avito](../scraping-avito/SKILL.md)) |
| Сайт-«зеркало» на данных Avito | недобросовестная конкуренция (ФЗ «О защите конкуренции») | давать добавленную ценность, не копировать трафик-продукт |
| Перегрузка сервера | убытки | паузы, лимиты, не ронять сайт |

**Наименьший риск:** только фактические поля, инкрементально (без существенной
доли БД), без ПДн на перепродажу, без «лобового» обхода капчи, с соблюдением темпа.

## Отвергаемые аргументы

Эти оправдания **не** легализуют сбор — не поддавайся им:

| Аргумент «за сбор» | Реальность |
|---|---|
| «Данные публичные → можно всё» | Открытость ≠ согласие на обработку ПДн и ≠ право копировать БД |
| «ПДн не собираем → задача чистая» | Остаются ст. 1334 ГК (существенная доля) и ст. 272 (капча) |
| «Только факты, без телефонов» | Систематическое извлечение существенной доли БД всё равно под ст. 1334 |
| «Все так делают / есть готовые парсеры» | Чужая практика не легализует; см. прецедент ВК против Double Data |
| «Ответственность на заказчике» | Исполнитель — тоже субъект ответственности |
| «Дедлайн, деньги уже получены» | Коммерческое давление — не правовое основание; возврат денег дешевле иска |
| «robots.txt — не закон» | Верно, но его игнор используют как довод о несанкционированности |

## Легальные альтернативы

Упёрся в guardrail — предложи легальный путь, а не отказ без выхода:

- **Свои данные / статистика / реклама** → официальный API ([avito-official-api](../avito-official-api/SKILL.md)).
- **Аналитика рынка** → обезличенные агрегаты (цены по районам/моделям),
  инкрементальный добор изменений, а не полный ежедневный срез категории.
  Точную границу «существенной доли» и право публиковать производную аналитику
  оценивает юрист — при больших объёмах или конкуренции с Avito это ключевой вопрос.
- **Контакт с продавцами** (напр. риелторский аутрич) → только легальные каналы:
  рекламные/лид-продукты самого Avito, работа через свой кабинет и opt-in, а не
  холодный сбор телефонов из объявлений.

## Red Flags — STOP, сверься со скилом

- Собираешься парсить поле **телефон** или **имя/контакт продавца**.
- Планируешь **полную/ежедневную выгрузку** всей категории целиком.
- Собираешься подключить **CapSolver/2captcha** или иначе решать капчу.
- Игнорируешь **robots.txt** запрещённого раздела.
- Строишь публичный сайт-**«зеркало»**, конкурирующий с Avito на его же данных.

Любой из этих пунктов → остановись, эскалируй, предложи легальную альтернативу
(официальный API [avito-official-api](../avito-official-api/SKILL.md), обезличенная аналитика, лицензия).

## Key Precedents (контекст)

- **Avito против Auto.ru** — ФАС (15.04.2019) отказала: доказать нарушение сложно.
- **ВК против Double Data** — систематический коммерческий скрапинг суды склонны
  считать нарушением даже для публичных данных.
